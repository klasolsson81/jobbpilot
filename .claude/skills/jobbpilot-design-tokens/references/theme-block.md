## shadcn/ui token mapping

After pasting the `@theme` block, configure `components.json` and
`tailwind.config.ts` to map shadcn's expected variable names to the
JobbPilot tokens. shadcn expects `--background`, `--foreground`, `--primary`,
`--ring` etc. — map these to the corresponding `--color-*` values.

Example mapping snippet for `tailwind.config.ts`:

```ts
// Minimal shadcn → JobbPilot token bridge
// Full version generated by: pnpm dlx shadcn@latest init
colors: {
  background:  'var(--color-surface-primary)',
  foreground:  'var(--color-text-primary)',
  primary:     { DEFAULT: 'var(--color-brand-600)', foreground: '#FFFFFF' },
  secondary:   { DEFAULT: 'var(--color-surface-secondary)', foreground: 'var(--color-text-primary)' },
  destructive: { DEFAULT: 'var(--color-danger-600)', foreground: '#FFFFFF' },
  border:      'var(--color-border-default)',
  input:       'var(--color-border-default)',
  ring:        'var(--color-brand-600)',
  muted:       { DEFAULT: 'var(--color-surface-tertiary)', foreground: 'var(--color-text-secondary)' },
}
```

Run `pnpm dlx shadcn@latest init` first to generate the base config, then
apply the token bridge above.
