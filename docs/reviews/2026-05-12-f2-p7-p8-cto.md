## Batteri 1 — F2-P7

### P7-A: Offset-based (A1)
KISS/YAGNI — UI-konsumenten är jobblista med "Sida 1, 2, 3". Cursor löser concurrent-insert-skew som inte existerar med 10-min-sync + `publishedAt DESC`. REP/CCP (Martin 2017 kap. 13) — `PagedResult<T>` finns redan etablerad genom TD-55.

### P7-B: Återanvänd generic PagedResult<T> (B1)
DRY (Hunt/Thomas 1999). Filen finns i `Application.Common`. TD-8 (GetPipeline) ärver senare utan refactor.

### P7-C: Whitelisted sort-enum (C2)
CLAUDE.md §5.1 "Magic strings förbjudet". OCP (Martin 2017 kap. 8) — enum för OCP-extension. Fält: `PublishedAt` (default), `ExpiresAt`. `CompanyName` när enum behövs.

### P7-D: Endast paginering (D1)
SRP per change-reason. Search/filter har egen UI-design-yta — lyfts som **TD-70** efter P7 (legitim per §9.6 — separat sub-fas inom Fas 2 + JobTech-search-param-spec saknas).

### P7-E: Breaking byte av befintlig endpoint (E1)
Ingen frontend-konsument finns idag (auth-gated, UI saknas). Inga produktionskonsumenter = ingen versions-skuld.

## Batteri 2 — F2-P8

### P8-A: Microsoft.Extensions.Http.Resilience + AddStandardResilienceHandler (A1)
Microsoft Learn officiell rekommendation .NET 10. `Microsoft.Extensions.Http.Polly` deprecated. Polly v8 är runtime — vi specificerar paketleverantör som BUILD.md lämnade öppen.

### P8-B: Hybrid Refit (JobSearch) + typed-client (JobStream) (B3)
SRP — två change-reasons (REST/JSON vs NDJSON-stream + polymorft event-schema). LSP via `IJobSource`-port.

### P8-C: Full BUILD.md-spec direkt (C2) — distribuerad i I3-sub-batches
Snapshot-only (C1) skapar interim-state. JobSearch-only (C3) är anti-mönster ("bulk discouraged — use Stream").

### P8-D: ExternalReference value object (D2)
CLAUDE.md §5.1 primitive obsession. Evans 2003 "Value Objects". Aggregate consistency boundary bevarad — JobAd är *en* annons oavsett källa.

### P8-E: UNIQUE-index + DbUpdateException-catch (E1)
CLAUDE.md §3.6 "använd IAppDbContext direkt" (E3 avvisad). Defense-in-depth + TOCTOU-skydd mot parallella Hangfire-workers.

### P8-F: JobAd.Archive() vid removal-event (F3)
DRY — befintlig metod, idempotent. DeletedAt är GDPR-cascade (fel semantik). Hard-delete förstör arbetsmarknad-historik.

### P8-G: Ingen cache mellan Hangfire-runs (G1)
DB är källan. Rate-limit (F2-P2) adresserar redan DoS. YAGNI utan mätdata.

### P8-H: PII-fri + sync-audit (H3)
`JobAdsSyncedDomainEvent(source, fetchedCount, addedCount, archivedCount, errorCount, ...)` — GDPR Art. 30 record of processing.

### P8-I: Tre sub-batches (I3)

| Batch | Scope |
|---|---|
| **P8a** | ADR 0032 + Domain (ExternalReference VO, JobAd.Import/UpdateFromSource, JobAdImportedDomainEvent). EF-migration (source+external_id+raw_payload+UNIQUE). Tester. |
| **P8b** | Infrastructure: IJobTechClient (Refit) + IJobTechStreamClient (typed) + PlatsbankenJobSource. Resilience-config. JobTechOptions. Admin-trigger-endpoint `POST /api/v1/admin/job-ads/sync/platsbanken`. WireMock-tester. |
| **P8c** | Hangfire SyncPlatsbankenJob (10min Stream + 02:00 Snapshot). JobAdsSyncedDomainEvent audit-wire. Dedup + removal-handling via Archive(). E2E. |
