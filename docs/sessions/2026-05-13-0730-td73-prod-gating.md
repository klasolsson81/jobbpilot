## CTO-rond 2026-05-13 (TD-73 prod-gating) — 13 beslut

1. **Q1 AggregateId-generering:** per-run-Guid via Hangfire jobId (OCP-väg framåt; DDD-identitet av "processed unit")
2. **Q2 Erasure-shape:** total null-out via `SetProperty(_ => null)` (KISS + GDPR Art. 5(1)(c) data-minimisation + Saltzer/Schroeder default-deny)
3. **Q3 Audit-granularitet:** EN aggregerad audit-rad per request (ADR 0024 D4-precedens + Martin CCP — handlingen är audit-yta, inte rad-count)
4. **Q4 RedactCmd.AggregateId:** per-request-Guid (RequestId) — följer Q3
5. **Q5 GIN-index:** defer till TD-76 (Beck YAGNI + Knuth premature optimization; F2-volym ~5-15k INSERTs/dygn netto)
6. **R-Risk1 Atomicitet:** best-effort + Hangfire retry + idempotens-check + Critical log (Fowler 2018 — distributed transaction över job-scope är över-spec)
7. **R-Risk2 Name-matching:** Email-only nu, Name → TD-75 (YAGNI + Art. 17 kräver inte name-identifier; email är primär)
8. **M1 ADR-shape:** ny ADR 0035 + amendment till ADR 0032 §8 + cross-ref ADR 0024 (Ford/Parsons/Kua immutability + Evans BC-disciplin)
9. **M2 Klas-STOPP-buntning:** INGET Klas-STOPP — entydiga principer i alla 13 frågor
10. **M3 Snapshot-shim:** SyncPlatsbankenSnapshotCommand har redan inte IAuditableCommand-marker — no-op
11. **M4 ICorrelationIdProvider Hangfire-scope:** impl-validation räcker (ADR 0022 specade redan stub-impl per Worker-scope)
12. **M5 SystemEventAuditor lifetime:** Scoped (matchar IAppDbContext)
13. **M6 Volym sanity-check:** GIN-defer korrekt — ~3-10k INSERTs/dygn realistic, GIN-overhead för intet

## Web-search (CLAUDE.md §9.5)

Verifierat 2026-05-13 innan implementation:

- **[Npgsql 10.0 Release Notes](https://www.npgsql.org/efcore/release-notes/10.0.html):** EF Core 10 + Npgsql 10 stöder `ExecuteUpdateAsync` med `SetProperty` på jsonb → translates till `jsonb_set`
- **[GitHub Issue #3745](https://github.com/npgsql/efcore.pg/issues/3745):** EF Core 10 har regression på `.Contains()` mot jsonb-mapped strings; `EF.Functions.JsonContains` (@>) är säker → vi använder den
- **[PostgreSQL Docs 18 — GIN](https://www.postgresql.org/docs/current/gin.html):** `jsonb_path_ops` mer performant för @>/@?/@@-operatorer
- **[pganalyze — GIN Index The Good and Bad](https://pganalyze.com/blog/gin-index):** write-overhead vid frekventa updates → bekräftar Q5=defer

## Disciplinmissar fångade + fixade

1. **Architect föreslog `EF.Functions.JsonContains` i Application-handler** — Clean Arch-brott (Npgsql i Application). Fixad genom att skapa `IRecruiterPiiPurger`-port i Application + Postgres-impl i Infrastructure. Samma mönster som `IAuditTrailEraser` (ADR 0024 D3).
2. **Architect+arch-test listade `RedactRecruiterPiiCommandHandler` som ISystemEventAuditor-konsument** — fel; handlern är `IAuditableCommand` + går via `AuditBehavior`. Fixad i arch-test (3-jobb-lista) + ADR 0035 §7 docs-not.
3. **Stream-job finally-block kunde maska originalexception vid audit-failure** (code-reviewer Minor-1). Fixad in-block med try/catch (CA1031-suppress + CA2219-disciplin) + Cwalina/Abrams §7.5-not. Snapshot/Purge har audit-wire post-completion, inte i finally — ingen samma risk.
4. **`JsonStringEnumConverter` saknades** för admin-endpoint enum-deserialisering — initialt 4 integration-tester failade på request body. Fixad via `[JsonConverter(typeof(JsonStringEnumConverter<RecruiterIdentifierType>))]` på enum (lokal annotation, inte global JSON-opts-skifte).
