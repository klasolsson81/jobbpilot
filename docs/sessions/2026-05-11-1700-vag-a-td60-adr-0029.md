## Beslut/avvägningar

1. **ADR 0029 är komplementär till 0028, inte supersession** — ADR 0028
   kärnbeslut oförändrade (A1, defense-in-depth, marker, bootstrap,
   konstant-separation). Bara claim-placering har flyttats. Ford/Parsons/Kua
   2017 ADR append-only-disciplin.
2. **Integration-test över unit-test för transformation-beteende** —
   `IClaimsTransformation` är pipeline-komponent, dess kontrakt är
   integration-nivå (Cohn 2009 Test Pyramid + Microsoft Learn
   "Integration tests in ASP.NET Core" `WebApplicationFactory`-pattern).
3. **Sentinel-claim-call-count-verifiering defererad** — kräver
   Infrastructure.UnitTests-projekt. YAGNI till nästa `IClaimsTransformation`
   tillkommer eller transformations-bug identifieras.
4. **0 nya TDs lyfta** — alla agent-review-fynd hanterade in-block per
   4h-regeln + CTO-avvisade Minor 1/2 från dotnet-architect som genuint inte
   TD-värda (cosmetic + spekulativ pattern-ADR).
5. **Scope-utökning från 45 min → 2.5h** — accepterad eftersom agent-review-
   driven inom samma uppdrag, inte ny feature. M-2 var falsk mitigerings-claim
   i Accepted-ADR om säkerhet — inte "polish", korrigering av ADR-integritet.
