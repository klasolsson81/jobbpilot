## Skill-doc-konformitet snarare än multi-approach-val

Båda TDs visade sig vara **dokumentations-drift**, inte design-val.
Skill-läsning klargjorde planen entydigt — ingen CTO-invocation krävdes:

**TD-54:** `jobbpilot-design-tokens` SKILL.md säger att `text-text-tertiary`
är för "Disabled, placeholder". `jobbpilot-design-a11y` SKILL.md §4 säger
explicit "text-tertiary fails for body text on surface-secondary — only
use it for decorative/non-essential text". Hela funktionella användningen
(timestamps, sekundärtext, ID-fragment) var drift från docs.

**TD-42:** `jobbpilot-design-components` SKILL.md säger "Input/Textarea/Select
Height: 36px (h-9) default, 32px (h-8) for dense contexts".
`jobbpilot-design-a11y` SKILL.md §9 säger "Button (default, h-9 = 36px) —
Use lg (h-11 = 44px) for critical CTAs". Kod-baseline (h-8 default + h-9 lg)
var pure drift från skill-docs.

CTO-invocation hoppade över eftersom valet var ren skill→kod-konvergens.
Klas-STOPP hoppade över eftersom planen följde befintlig dokumentation.

## Lärdomar

- **Skill-docs är auktoritativa när kod är drift.** Båda TDs visade sig vara
  kod-drift från skill-docs, inte design-val. Skill-läsning innan
  CTO-invocation = effektivare flöde när planen ändå är entydig.

- **Kontextuell mapping > strikt substitution.** TD-54 hade lockat
  approach att byta ut alla 16 träffar för enkelhets skull. Men funktionell
  vs dekorativ är meaningful distinction — bevarad tertiary för separatorer
  matchar både a11y-skill §4-undantaget och components-skill breadcrumb-pattern.

- **In-block-fixar för 4h-regel-konformitet inkluderar follow-up-konsekvenser
  av primitive-ändringar.** TD-42 M2 (page-header CTAs `size="sm"` inkonsekvent
  efter default-uppgradering) var inte original-scope men direkt konsekvens
  av samma h-byte. Fixad in-block per "samma touch"-disciplin.

- **Civic-utility-densitet vinner över WCAG 2.5.5 AAA strikt.** h-9 (36px)
  default + h-11 (44px) för critical CTAs matchar Stripe Dashboard / GOV.UK
  / 1177 desktop-utility-pattern. WCAG 2.5.5 är AAA (rekommendation, inte
  AA-krav). Bevarade dense-context-varianter (xs/sm) visar disciplinerad
  förståelse — inte "alla knappar måste vara 44px".

- **Native form-control-divergence är systemiskt mönster.** TD-57 (lyft från
  TD-42 M3) avslöjar att vi har minst 2 native-form-element som inte ärver
  Input-primitive-defaults. Lift av inputBaseClasses till shared module är
  förmodligen rätt långsiktig lösning, men behöver designgenomgång.
