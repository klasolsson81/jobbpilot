## 9. In-block-fix-batch FÖRE v0.2-tag (Klas-GO pending)

Per CTO-rond 2026-05-13. Tre konkreta leveranser:

### 9.1 CloudWatch-alarm: JobTech-sync 3 consecutive failures

- **Metric filter** mot `/aws/ecs/jobbpilot-dev/worker` (och prod-motsvarighet): pattern matchar Worker-error-logs från `SyncPlatsbankenStreamJob` / `SyncPlatsbankenSnapshotJob` med `errors > 0`
- **Alarm:** `Sum >= 3` över `30 min` (3 cron-tick × 10 min)
- **SNS-topic:** befintlig `secops-anomaly` eller ny `operations-alarms` (CTO-fråga)
- **Terraform:** utökning av `modules/cloudwatch_security_alarms` eller ny `cloudwatch_operations_alarms`-modul (samma precedens som TD-68)
- **Adresserar:** BUILD.md §14.4 + Fas 2-aktiv integration utan observability-täckning

### 9.2 CloudWatch-alarm: SystemEventAuditor failure (EventId 5602)

- **Metric filter** mot `/aws/ecs/jobbpilot-dev/{api,worker}`: pattern matchar `level="Critical" AND EventId.Id=5602` (eller motsvarande LoggerMessage-template)
- **Alarm:** `Sum >= 1` över `5 min` (varje failure är signal — best-effort kräver alarm-wire)
- **Adresserar:** ADR 0035 §6 egen leveransspec ("Critical-log → CloudWatch-alarm efter retry-exhaustion")
- **GDPR-kongruens:** utan alarm är audit-failure tyst → Art. 30 record-of-processing kan kompromissas tyst

### 9.3 RDS backup-retention 7d → 14d (prod-Terraform)

- **Endast prod-env-tfvars** ändras: `rds_backup_retention_days = 14`
- **Dev oförändrat** (7d räcker för dev-lifecycle)
- **Dokumentera** i `docs/runbooks/aws-setup.md` eller motsvarande: "14d retention = EDPB CEF 2025-verifierad acceptabel Art. 17 backup-overlap; industry-common; valt över 35d-max för att inte öka GDPR-yta utan TD-13"

**Scope-uppskattning:** 2-3 commits, ~3-4h CC-tid (Terraform + ADR-cross-ref + test).

**Klas-STOPP-flagga:** JA, FÖRE batch-leverans + FÖRE v0.2-tag-push.

## 10. Sluttillstånd för v0.2-tag-readiness

När §9.1 + §9.2 + §9.3 är klara + Klas verifierat 7.4 + 7.5:

```bash
git tag v0.2 <HEAD>
git push origin v0.2
# triggar deploy-prod.yml → manuell approval → ECS-deploy + migrations-task
```

Förvänta: prod-deploy-pipelinen körs **för första gången**. Verifiering:

- ECS-tasks startar (api + worker)
- `/api/ready` returnerar 200 mot prod-domän
- Hangfire-jobben kör enligt schema (`*/10` cron)
- Audit-wire bevisas via prod-CloudWatch (`System.JobAdsSynced`-INSERT)
- CloudWatch-alarms 9.1 + 9.2 i `OK`-state

Vid issues: rollback via `aws ecs update-service --task-definition <previous>` (BUILD.md §15.4).
